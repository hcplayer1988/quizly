# Quizly - AI-Powered Quiz Generator

A Django REST Framework backend that converts YouTube videos into AI-generated quizzes using OpenAI Whisper for transcription and Google Gemini for quiz generation. Features JWT-based cookie authentication and a full quiz management system.

## Features

- **JWT Cookie Authentication**: Secure httpOnly cookie-based token management
- **Token Blacklisting**: Logout invalidates refresh tokens permanently
- **YouTube Integration**: Download audio from any public YouTube video via yt-dlp
- **AI Transcription**: Automatic speech-to-text using OpenAI Whisper
- **AI Quiz Generation**: 10-question multiple-choice quizzes generated by Google Gemini
- **Quiz Management**: Full CRUD for quizzes with user ownership enforcement
- **CORS Enabled**: Ready for frontend integration

## Tech Stack

- **Django 6.0.3**
- **Django REST Framework**
- **SimpleJWT** for JWT authentication with token blacklisting
- **yt-dlp** for YouTube audio downloading
- **yt-dlp-ejs** for JavaScript challenge solving
- **Deno** runtime for yt-dlp JS challenge support
- **OpenAI Whisper** for audio transcription
- **Google Gemini** (`google-genai`) for quiz generation
- **SQLite Database** (easily switchable to PostgreSQL)
- **django-cors-headers** for CORS support
- **python-decouple** for environment variables

## Installation & Setup

### Prerequisites

- Python 3.10 or higher
- pip (Python package manager)
- Virtual environment (recommended)
- FFmpeg installed and available in PATH
- Deno runtime (required for yt-dlp JS challenge solving)

**Install FFmpeg (Windows):**
Download from https://ffmpeg.org/download.html and add to system PATH.

**Install Deno (Windows):**
```bash
winget install DenoLand.Deno
```
After installation, add Deno to your system PATH permanently via Windows Environment Variables.

### 1. Clone the Repository

```bash
git clone <repository-url>
cd quizly
```

### 2. Create Virtual Environment

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux/Mac
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

**Requirements:**
```txt
Django==6.0.3
djangorestframework==3.16.1
djangorestframework_simplejwt==5.5.1
django-cors-headers==4.9.0
google-genai==1.66.0
openai-whisper==20250625
python-decouple==3.8
yt-dlp==2026.3.3
yt-dlp-ejs==0.5.0
```

### 4. Environment Configuration

Create a `.env` file in the project root (next to `manage.py`):

```env
SECRET_KEY="your-secure-secret-key-here"
GEMINI_API_KEY=your-gemini-api-key-here
DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost
STATIC_ROOT=staticfiles
MEDIA_ROOT=media
```

**Get your Gemini API Key:**
1. Go to https://aistudio.google.com/apikey
2. Create a new project
3. Generate an API key
4. Paste it into `.env`

**Generate a secure SECRET_KEY:**
```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

**IMPORTANT**: Never commit the `.env` file to version control!

### 5. Database Setup

```bash
python manage.py migrate
```

### 6. Create Superuser (Optional)

```bash
python manage.py createsuperuser
```

### 7. Start Development Server

**IMPORTANT**: Deno must be available in PATH before starting the server. On Windows, set the PATH in the same terminal session if not permanently configured:

```bash
# Windows (temporary PATH - only needed if Deno not in system PATH)
$env:PATH += ";C:\path\to\deno"

# Then start server
python manage.py runserver
```

The API will be available at: `http://127.0.0.1:8000/`

## Configuration

### Important Settings

#### CORS Configuration (`core/settings.py`)

Update `CORS_ALLOWED_ORIGINS` with your frontend URL:

```python
CORS_ALLOWED_ORIGINS = [
    'http://127.0.0.1:5500',
    'http://localhost:5500',
]

CORS_ALLOW_CREDENTIALS = True
```

#### JWT Cookie Settings

```python
SIMPLE_JWT = {
    'ACCESS_TOKEN_LIFETIME': timedelta(minutes=30),
    'REFRESH_TOKEN_LIFETIME': timedelta(days=7),
    'ROTATE_REFRESH_TOKENS': True,
    'BLACKLIST_AFTER_ROTATION': True,
    'AUTH_COOKIE': 'access_token',
    'AUTH_COOKIE_SECURE': True,
    'AUTH_COOKIE_HTTP_ONLY': True,
    'AUTH_COOKIE_SAMESITE': 'Lax',
}
```

#### REST Framework Settings

```python
REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework_simplejwt.authentication.JWTAuthentication',
        'auth_app.api.permissions.CookieJWTAuthentication',
    ),
    'DEFAULT_PERMISSION_CLASSES': (
        'rest_framework.permissions.IsAuthenticated',
    ),
}
```

## API Endpoints

### Authentication

| Method | Endpoint | Description | Auth Required |
|--------|----------|-------------|---------------|
| POST | `/api/register/` | Register new user | No |
| POST | `/api/login/` | Login and set JWT cookies | No |
| POST | `/api/logout/` | Logout and blacklist token | Yes |
| POST | `/api/token/refresh/` | Refresh access token | No (cookie) |

### Quiz Management

| Method | Endpoint | Description | Auth Required |
|--------|----------|-------------|---------------|
| GET | `/api/quizzes/` | List own quizzes | Yes |
| POST | `/api/quizzes/` | Create quiz from YouTube URL | Yes |
| GET | `/api/quizzes/{id}/` | Get quiz detail | Yes (Owner) |
| PATCH | `/api/quizzes/{id}/` | Update quiz title/description | Yes (Owner) |
| DELETE | `/api/quizzes/{id}/` | Delete quiz | Yes (Owner) |

## Special Features & Considerations

### 1. JWT Cookie Authentication

Tokens are stored as httpOnly cookies — not in localStorage. This means:

- The browser sends cookies automatically with every request
- JavaScript cannot access the tokens (XSS protection)
- `CORS_ALLOW_CREDENTIALS = True` must be set
- The frontend must use `credentials: 'include'` in fetch requests:

```javascript
const response = await fetch('http://127.0.0.1:8000/api/quizzes/', {
  method: 'GET',
  credentials: 'include',
});
```

### 2. YouTube Audio Processing

Quiz creation follows this pipeline:

1. Extract video ID from the YouTube URL
2. Download audio using yt-dlp with FFmpeg postprocessing
3. Convert audio to MP3 format
4. Transcribe with Whisper
5. Generate quiz with Gemini
6. Save quiz and questions to database

**Supported URL formats:**
- `https://www.youtube.com/watch?v=VIDEO_ID`
- `https://youtu.be/VIDEO_ID`

**Only public YouTube videos are supported.** Age-restricted, private, or region-blocked videos will fail.

### 3. Whisper Transcription

Whisper runs locally on CPU. This means:

- No additional API costs
- Transcription is slower than GPU (expect 1-3 minutes for a 10-minute video)
- A warning `FP16 is not supported on CPU; using FP32 instead` is normal and can be ignored
- The `base` model is used by default (good balance of speed and accuracy)

### 4. Gemini Quiz Generation

The quiz is generated using `gemini-2.5-flash`. The prompt enforces:

- Exactly 10 questions per quiz
- Exactly 4 answer options per question
- One correct answer per question
- Valid JSON output only

**Free Tier Limits** (per day per project):
- 1,500 requests/day
- 1,000,000 tokens/minute

If you hit the daily limit, wait until midnight UTC or create a new Google Cloud project with a fresh API key.

### 5. Deno & yt-dlp JS Challenge Solving

YouTube uses JavaScript challenges to protect against bots. yt-dlp requires the `yt-dlp-ejs` package and the Deno runtime to solve these challenges.

**Setup:**

1. Install `yt-dlp-ejs`:
```bash
pip install yt-dlp-ejs
```

2. Install Deno and add it to your **system PATH permanently** (Windows):
   - Open: System Properties → Environment Variables → Path → New
   - Add the path to your Deno executable directory

3. Verify Deno is available:
```bash
deno --version
```

Without Deno in PATH, yt-dlp will fail with format errors.

### 6. CORS Configuration

**CRITICAL**: Disable any "Allow CORS" browser extensions!

The backend has proper CORS configured. Browser extensions interfere and cause errors.

**Common CORS issues:**
- Browser extension blocking requests
- Frontend running on different port than configured
- Missing `credentials: 'include'` in frontend fetch calls

**Solution:**
1. Disable ALL CORS browser extensions
2. Update `CORS_ALLOWED_ORIGINS` in `settings.py`
3. Clear browser cache (Ctrl + Shift + R)
4. Test in Incognito mode

## Troubleshooting

### Issue: yt-dlp download fails

**Symptoms:**
```
ERROR: [youtube] VIDEO_ID: Video unavailable
WARNING: n challenge solving failed
```

**Solution:**
1. Ensure Deno is installed and available in PATH
2. Ensure `yt-dlp-ejs` is installed: `pip install yt-dlp-ejs`
3. Restart the terminal and server after adding Deno to PATH
4. Try a different public YouTube video
5. Update yt-dlp: `pip install -U yt-dlp`

### Issue: Gemini API quota exceeded

**Symptoms:**
```json
{"detail": "Error generating quiz: 429 RESOURCE_EXHAUSTED..."}
```

**Solution:**
1. Wait until midnight UTC (daily limit resets)
2. Create a new Google Cloud project with a fresh API key
3. Activate billing on the project (limits increase significantly)
4. Try a shorter video (fewer tokens used)

**Note:** All keys in the same Google Cloud project share the same quota.

### Issue: Whisper FP16 warning

**Symptoms:**
```
UserWarning: FP16 is not supported on CPU; using FP32 instead
```

**This is not an error.** Whisper runs in FP32 mode on CPU automatically. Transcription will still work correctly, just slightly slower.

### Issue: Audio file cannot be read

**Symptoms:**
```json
{"detail": "Error processing video: Failed to load audio..."}
```

**Solution:**
1. Ensure FFmpeg is installed and in PATH: `ffmpeg -version`
2. The video might use a streaming format (HLS/m3u8) — FFmpeg postprocessing converts it to MP3 automatically
3. Try a different video

### Issue: 401 Unauthorized on endpoints

**Solution:**
1. Ensure you are logged in and cookies are set
2. Check that `credentials: 'include'` is set in frontend fetch calls
3. Access token may have expired — call `POST /api/token/refresh/` to get a new one
4. If refresh token is also expired, log in again

## Production Deployment

### Before deploying:

1. **Set DEBUG = False** in `.env`:
```env
DEBUG=False
```

2. **Set secure ALLOWED_HOSTS** in `.env`:
```env
ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com
```

3. **Enable HTTPS cookie settings** in `settings.py`:
```python
SIMPLE_JWT = {
    ...
    'AUTH_COOKIE_SECURE': True,   # Already True - requires HTTPS
    'AUTH_COOKIE_SAMESITE': 'Lax',
}
```

4. **Update CORS** in `settings.py`:
```python
CORS_ALLOWED_ORIGINS = [
    'https://yourdomain.com',
]
```

5. **Use PostgreSQL** instead of SQLite:
```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': config('DB_NAME'),
        'USER': config('DB_USER'),
        'PASSWORD': config('DB_PASSWORD'),
        'HOST': config('DB_HOST'),
        'PORT': '5432',
    }
}
```

6. **Collect static files**:
```bash
python manage.py collectstatic
```

7. **Use a production WSGI/ASGI server** (e.g. Gunicorn):
```bash
pip install gunicorn
gunicorn core.wsgi:application
```
